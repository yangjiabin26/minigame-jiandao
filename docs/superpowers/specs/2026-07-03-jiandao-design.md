# 《剑道》抖音小游戏设计文档

- 日期：2026-07-03
- 状态：已确认
- 分类：动作小游戏 / H5游戏 / 战士 / 闯关
- 适龄提示：16 岁以上

## 一、概述

玩家扮演一名身披重甲的骑士，在山谷、森林和城堡中闯关战斗，击败敌人与 Boss，通过金币升级武器、盔甲和闪避能力，最终击败黑骑士完成冠军之路。

**核心决策**（头脑风暴确认）：

| 决策项 | 结论 |
|---|---|
| 操作方式 | 虚拟摇杆（左半屏动态）+ 攻击/闪避按钮（右下角） |
| 视角形式 | 2D 顶视角（俯视/45° 斜视角），竖屏 |
| 美术资源 | 免费素材库（Kenney、itch.io 等 CC0 像素素材） |
| 内容范围 | 精简版：8 关 + 三线升级系统 |
| 技术方案 | 原生 JS + Canvas 2D，不用游戏引擎 |
| 变现方式 | 激励视频广告（三个点位），无内购 |
| 后端 | 无服务器，本地存档 |
| 开发者账号 | 从零注册（个人主体） |

## 二、项目结构与架构

```
minigame-jiandao/               # 游戏项目根目录
├── game.js                     # 抖音小游戏入口
├── game.json                   # 全局配置（竖屏等）
├── project.config.json         # 开发者工具项目配置
├── src/
│   ├── main.js                 # 游戏主类：循环、场景切换
│   ├── core/                   # 轻量框架层
│   │   ├── loop.js             # requestAnimationFrame 游戏循环
│   │   ├── input.js            # 触摸输入 → 虚拟摇杆/按钮
│   │   ├── camera.js           # 跟随镜头
│   │   ├── collision.js        # 圆形/矩形碰撞
│   │   ├── sprite.js           # 精灵图帧动画
│   │   ├── audio.js            # 音效管理（tt.createInnerAudioContext）
│   │   └── storage.js          # 存档（tt.setStorageSync 封装）
│   ├── scenes/                 # 加载/主菜单/选关/战斗/结算
│   ├── entities/               # 玩家、敌人、Boss、飞行物、掉落物
│   ├── ui/                     # 虚拟摇杆、按钮、血条、弹窗
│   ├── data/                   # 关卡配置、敌人数值、升级数值
│   └── platform/               # 抖音 API 封装：广告、分享、录屏、存储
└── assets/                     # 精灵表、音效（总计 ≤ 2MB）
```

**架构要点**：

- **场景状态机**：加载 → 主菜单 → 选关 → 战斗 → 结算。每个场景独立管理更新与渲染，切换时清理干净。
- **实体组合**：玩家/敌人共享「移动 + 血量 + 碰撞 + 动画」基础实体类；敌人 AI 用状态机（巡逻 → 追击 → 攻击）。
- **数据驱动关卡**：每关一份配置（地图尺寸、墙体、刷怪点、出口位置），加关卡只加配置不改代码。
- **平台层隔离**：所有 `tt.` API 集中在 `platform/`，并提供浏览器 mock 实现——游戏可在浏览器中完整试玩调试，也便于日后移植微信小游戏。

## 三、核心玩法

### 操作（竖屏）

- 左半屏任意位置按下出现动态虚拟摇杆，八向移动。
- 右下角按钮：攻击（大）、闪避（小）。闪避向移动方向冲刺，0.3 秒无敌帧，2 秒冷却。

### 战斗

- 近战挥剑：攻击键在面朝方向产生 90° 扇形判定区，命中造成伤害 + 击退。
- 三连击：连按最多三段，第三击伤害更高。
- 打击感：屏幕微震 + 命中闪白 + 音效。
- 玩家受击后 0.5 秒无敌帧。

### 敌人（4 种 + 2 Boss）

| 敌人 | 行为 |
|---|---|
| 敌兵（近战） | 巡逻 → 发现玩家追击 → 近身挥刀 |
| 弓箭手 | 保持距离，直线射箭，箭可闪避 |
| 盾兵 | 正面免伤，需绕背攻击 |
| 狂战士 | 血量低于一半后加速冲撞 |
| 山谷督军（第 4 关 Boss） | 横扫 + 冲锋，技能有前摇提示 |
| 黑骑士（第 8 关 Boss） | 三阶段，技能组合逐阶段升级 |

### 关卡（8 关）

- 场景分布：山谷（1-2）→ 森林（3-4，第 4 关 Boss）→ 城堡（5-8，第 8 关最终 Boss）。三套地图贴图皮肤。
- 每关目标：清空敌人后出口开启，走到出口过关。房间式地图约 2-3 屏，镜头跟随。
- 死亡：可看激励视频原地满血复活（每关一次），否则回选关页。

## 四、升级系统、经济与广告

### 金币来源

击杀掉落（自动吸附拾取）、通关奖励、Boss 大额掉落。

### 三线升级（主菜单「铁匠铺」，全局生效）

| 升级线 | 效果 | 等级数 |
|---|---|---|
| 武器（剑） | 每级 +攻击力 | 8 级 |
| 盔甲 | 每级 +最大生命 | 8 级 |
| 闪避 | 减冷却、加距离 | 5 级 |

数值曲线：正常打法在第 4 关和第 6 关左右感到吃力，需回刷旧关升级 1-2 次——既是留存钩子也是广告点位价值来源。

### 激励视频广告（三个点位，均为用户主动触发）

1. **原地复活**：死亡时看广告满血复活，每关限一次。
2. **金币双倍**：结算页看广告本关金币 ×2。
3. **铁匠铺折扣**：看广告获一次升级半价，每日 3 次。

### 分享

通关 Boss 后弹「炫耀战绩」分享按钮（`tt.shareAppMessage`），纯增益不强制。

### 存档

金币、升级等级、关卡进度经 `storage.js` 存 `tt.setStorageSync`，带版本号字段支持后续迁移。无服务器。

## 五、数据流、错误处理与性能

### 数据流

单向流动：`input.js` 采集触摸 → 场景 update（实体逻辑、碰撞、数值结算）→ render 绘制。全局唯一 `GameState`（金币、升级、进度），改动后立即写存档。

### 错误处理

- 存档损坏 → 回退全新存档，不崩溃。
- 广告加载失败 → 降级策略：复活点位显示「本关不可复活」，双倍/折扣按钮隐藏；绝不因广告卡死流程（审核重点）。
- 资源加载失败 → 加载页重试按钮。
- `tt.onError` 上报未捕获异常，配合开发者后台错误监控。

### 性能目标

- 中低端安卓 60fps，最低接受 30fps。
- 图片合成精灵表；箭矢/金币/飘字用对象池；同屏敌人上限 12 个，超出的刷怪点延迟激活。
- 主包 ≤ 2MB（平台上限 4MB，留余量）。

## 六、测试策略

- **单元测试**（Node 可跑，TDD）：碰撞函数、伤害计算、升级数值曲线、存档读写与版本迁移。
- **浏览器 mock 试玩**：`platform/` 的 mock 实现让游戏在浏览器完整可玩，用于开发自测。
- **真机验收**（人工）：抖音开发者工具 + 真机预览，验收摇杆手感、按钮尺寸、帧率。

## 七、上线运营路线图

### 阶段一：开发

按本设计完成游戏本体；真机手感验收由用户负责。

### 阶段二：账号与资质（用户操作，附逐步指引）

1. 注册抖音开放平台账号（个人主体、实名认证），创建小游戏获得 AppID。
2. 软著：提审需《计算机软件著作权登记证书》，个人申请约 200-300 元、周期 1-2 个月（可加急）。开发过半即提前申请，避免成为上线卡点。
3. 版号：纯广告变现、无内购的休闲小游戏走备案通道，无需版号（提审前核对最新政策）。
4. 开通流量主（激励视频广告）权限。

### 阶段三：提审上线

- 材料：游戏名「剑道」、简介、截图、类目（动作）、适龄提示（16+）、自审自查报告。
- 开发者工具上传 → 提审 → 发布。设计已规避常见拒审原因：广告诱导点击、缺适龄提示、加载超时。

### 阶段四：运营迭代

- **数据**：开发者后台看次日留存、广告曝光/eCPM、关卡流失漏斗；哪关流失高调哪关难度。
- **获量**：接入游戏发行人计划（达人视频挂载，按转化付费）；游戏内录屏分享（`tt.getGameRecorderManager`）。
- **迭代**：首版验证后，第二版加 9-15 关新内容 + 新敌人；视收入考虑企业主体与内购。

## 八、非目标（首版不做）

- 横屏模式
- 多武器切换、装备部位系统
